Médio aceita o cadastro de até 50 agendamentos no mês, e custa R$79,90
Grande aceita o cadastro de até 100 agendamentos no mês, e custa R$149,90
Super aceita o cadastro de até 250 agendamentos no mês, e custa R$249,90

Todos os planos incluem aviso por email, sms ou whatsapp.

Após o limite de agendamentos esgotar, deve mostrar uma mensagem de sistema indisponível na página pública da empresa

Deve haver uma área no painel da empresa para ela administrar a assinatura do plano, ver as faturas pagas, a vencer ou vencidas, realizar pagamento ou upgrade.

1. Docker: não está instalado na máquina (docker not found). Instale Docker
   Desktop ou rode MySQL local.
2. Firebase: criar projeto, habilitar Email/Password, gerar service account
   JSON → secrets/firebase-service-account.json, anotar Web API key e Project ID
   no .env.
3. cp .env.example .env e preencher.
4. npm run migration:run → cria as tabelas.
5. npm run start → API em :3000, web em :4200.

  guard

  - Service — apps/api/src/modules/services/service.entity.ts (duração, buffer,

  - BusinessHour — dayOfWeek 0-6 + HH:mm
  - BusinessException — feriados e bloqueios pontuais (fullDay ou janela)

  business_exceptions com FKs e índices
- Endpoints CRUD tenant-isolados:
  - GET/PATCH /api/company (settings + slug com checagem de unicidade + toggles
    de notificação)

  - GET/PUT /api/company/business-hours (PUT substitui todos em transação;
    valida overlap em validateNoOverlap)
  - GET/POST/DELETE /api/company/business-exceptions

  (HH:mm, ordem, fullDay, datas)
